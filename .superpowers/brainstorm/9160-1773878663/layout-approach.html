Guide Hub Page — Layout Approach

How should the guides be organized on the page?

A

Category Cards with Expandable Sections

Big visual cards for each category (Buyer Guides, Seller Guides, Mortgage, etc.) — click a category to expand and see all guides inside it. Clean and uncluttered. Visitors pick their persona first, then browse.

Best for: Sites with lots of guides (you have 60+). Keeps the page from being overwhelming.
SEO note: All links render on page load for crawlers, just visually collapsed for users.
B

Full Grid — Everything Visible

All guides shown in a categorized grid. Each category is an H2 section with guide cards underneath. No collapsing — the full library is visible on one scroll. Sticky sidebar nav lets visitors jump between sections.

Best for: SEO power — every link and description visible on the page. Users can scan everything at once.
Tradeoff: Long page, but the sticky sidebar nav keeps it navigable.
C

Persona-First Landing (Tabs or Buttons)

Start with "I am a..." buttons: First-Time Buyer, Seller, Investor, Relocator, Military. Each filters the page to show only relevant guides. A "Show All" option reveals everything.

Best for: Lead conversion — visitors self-identify their persona, which you can track. Feels personalized.
Tradeoff: Requires JavaScript filtering. Some guides fit multiple personas.
My recommendation: Option B (Full Grid with Sticky Sidebar). With 60+ guides, the SEO value of having every link visible is massive. The sticky sidebar keeps it usable. This is the format that top real estate sites like Redfin and Zillow use for their resource centers.
